# MVP4 — メモ編集＋永続化（素朴パネル）

> 対象コード: [src/sessionLoader.ts](../../src/sessionLoader.ts)（安定キー）/ [src/extension.ts](../../src/extension.ts)（保存とパネル）
> 関連: [ROADMAP.md MVP4](../../../docs/ROADMAP.md)

## 概要

ノードに主観メモとピン留めを付けられるようにし、VS Code を再起動しても残るよう永続化する。
コアコンセプト（「あの良い状態のセッション」を後で見つける）に直結する最優先機能。
リッチな右パネルは作らず、メモ欄＋ピン＋保存＋開くボタンだけの素朴な下部バーにする（豪華版は MVP5）。

## 実装手順

### 1. ノードに「安定キー」を持たせる（sessionLoader.ts）
- 目的: メモの保存キーを決める。**ノード id（canonical uuid）は使えない** —
  代表レコードは「最初に読んだ個体」で決まり、ファイルの mtime 順が変わると別の uuid になるため、
  id でメモを保存すると再読込でメモが迷子になる
- やること: `stableNodeKey()` = canonical key（timestamp+type+内容指紋）の SHA-1 先頭 16 桁。
  canonical 化と同じ「論理イベントの同一性」なので、再起動・Fork 複製・読み込み順をまたいで不変
- 検証: 2 回読み込んで全 72 キーが一致・衝突 0 を確認

### 2. globalState への保存（extension.ts）
- 目的: VS Code 再起動後も残す（MVP4 完了条件）
- やること: `forkGraph.notes` キーに `{ [安定キー]: { memo, pinned, updatedAt } }` のマップを保存。
  空メモ＆ピン無しは削除して肥大化を防ぐ
- workspaceState でなく globalState の理由: フォールバックで別プロジェクトの森も表示でき、
  キー自体がプロジェクト非依存で一意なため

### 3. 素朴なメモパネル（Webview）
- クリック = ノード選択 → 下部バーに「プロンプト先頭／メモ欄／📌ピン／保存／セッションを開く」
- **ダブルクリック = セッションを開く**（MVP3 の機能はここに移動。シングルクリックは選択に変更）
- ピンはクリックで即保存。メモは「保存」ボタンか Ctrl+Enter
- グラフ上の表示: ピン留め = 📌、メモあり = ✎ をピル脇に描画（保存時に全再描画で反映）

## 判断理由

- **クリックの意味を変えた**: MVP3 では「クリック=開く」だったが、メモ編集（選択）と開くを
  両立させる必要があり、誤爆しやすい「開く」をダブルクリックに退避した
- **保存は明示ボタン**: 自動保存はタイミング起因の混乱（書き途中の保存等）を生むため、
  素朴版では明示保存＋Ctrl+Enter のみとした

## 検証

- `npm run compile` 通過、安定キー 72/72 一致・衝突 0
- 完了条件「メモ・ピンを書き換えて VS Code を再起動しても残っている」は実機で確認する
